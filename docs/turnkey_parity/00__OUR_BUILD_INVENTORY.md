# PaySpyre Backend ‚Äî Capability Inventory (as of 2026-07-10, main)

> Generated by repo sweep to serve as the "our side" baseline for the Turnkey Lender parity gap analysis
> (see `GAP_ANALYSIS.md` and the per-workplace video inventories `01__‚Ä¶` through `11__‚Ä¶` in this directory).

Stack: FastAPI (Python), SQLAlchemy, Alembic, PostgreSQL (JSONB-heavy), event-sourced audit via `platform_events`. All money is integer cents. Canada-only, currently dental-focused (verticals: dental/auto/veterinary defined). "One codebase, two modes" ‚Äî Simulation (mock vendors) vs Live ‚Äî switched by `USE_REAL_ADAPTERS` / `USE_REAL_NOTIFICATIONS` flags.

Four separately-mounted API surfaces plus metrics:
- `/api/v1` ‚Äî internal / admin cockpit (`app/api/v1/`)
- `/api/applicant/v1` ‚Äî borrower/applicant-facing (`app/api/applicant/`)
- `/api/clinic/v1` ‚Äî vendor/clinic-facing (`app/api/clinic/`)
- `/api/webhooks/v1` ‚Äî vendor webhooks (`app/api/webhooks/`)
- `/metrics` ‚Äî Prometheus (`app/api/metrics.py`)

Top-level routes: `GET /`, `GET /health`, `GET /csrf-token`.

## 2. Loan Lifecycle

### Application statuses (`platform_application_status`)
`started ‚Üí origination ‚Üí verifying ‚Üí pre_qualified ‚Üí awaiting_hard_pull ‚Üí underwriting / under_review ‚Üí approved / declined / withdrawn / expired`. (Migration 043 canonical schema; `under_review` = automated manual-review sink, distinct from `underwriting`.)

### Loan statuses (`platform_loan_status`)
`pending_disbursement ‚Üí active ‚Üí paid_off / delinquent / charged_off / cancelled`.
- Agreement sub-status: `not_sent ‚Üí sent ‚Üí signed / declined` (SignNow).
- Disbursement sub-status: `not_started ‚Üí in_progress ‚Üí completed / failed` (Zumrails).
- Schedule item status: `scheduled / paid / partial / late / waived`.
- Verification status: `pending / in_progress / passed / failed / expired / manual_review`.

### Decisioning (`app/services/flow_engine.py` ‚Äî pure functional core)
Bands (default, overridable per-product via `verification_matrix.bureau.manual_review_band`): **score ‚â• 680 ‚Üí approved; 600‚Äì679 ‚Üí manual_review; < 600 ‚Üí declined.** Same band powers widget pre-qual.
- Precedence: hard disqualifiers (active bankruptcy, sub-floor score, Quebec gate) are final declines. Soft/indeterminate conditions ‚Üí manual_review.
- Verification sequence via product `verification_matrix`: identity (KYC min_confidence) ‚Üí income (bank link, optional) ‚Üí bureau soft pull ‚Üí conditional hard pull. NSF count (`nsf_count_90d`) captured as a signal, not a hard rule.
- Stable `decision_reasons` codes (quebec_coming_soon, manual_review_band, bureau_below_minimum, active_bankruptcy, fraud_signal_review, identity_manual_review, verification_failed:*, verification_unknown:*).
- Co-applicant support: group combine via "average" score strategy (`applicant_role` primary/co_applicant). `decision_ruleset` references YAML rule files.
- Orchestration (`flow_orchestrator.py`): persists transitions/events; decisions run against an **immutable product config snapshot** (migration 026), not the live product row.

### Payment scheduling (`loan_servicing.py`)
- `generate_amortization_schedule` ‚Äî equal-payment fully-amortizing, exact integer-cent arithmetic, rounding remainder into final installment.
- `create_loan_from_application` ‚Äî books loan + schedule (one loan per application, DB-enforced).
- `record_payment` ‚Äî oldest-unpaid-installment-first; delinquency cure; idempotent on `(loan_id, external_ref)`.
- `generate_statement` ‚Äî periodic billing statements.
- APR/quote engine (`loan_quote.py`): Canadian regulatory APR per SOR/2001-104 s.3-4, criminal-rate cap check, fees from `pricing_config["fees_cents"]`.

### Collections / Dunning (`services/dunning.py`, `notification_config.py`)
- `DunningPolicy`: reminders 7 & 2 days before due; overdue escalation at 7/14/21/30/60/90 days after; beyond 90d manual. Email-only currently (SMS planned).
- **Late fees explicitly DISABLED** (`late_fee_enabled=False`) ‚Äî Canadian provincial law.
- Cadence/channels DB-backed (`platform_notification_rules`) with typed fallback. Idempotent per `(schedule_item, milestone)`. Province send windows enforced.

### Payoffs / NSF / Write-offs
- **Payoff:** `compute_payoff` (remaining principal + accrued interest); admin `GET /loans/{id}/payoff-quote`.
- **NSF:** `nsf_count_90d` bank signal; failed collections ‚Üí `loan_payment_failed` event only ‚Äî **no automatic NSF-retry/fee state machine.**
- **Write-off/charge-off:** `charged_off` status; maker-checker only. Aging job never disturbs terminal statuses.

## 4. Documents / E-sign, Notifications, Reporting

**Documents & e-sign:**
- KYC/doc upload (`document_storage.py`) ‚Äî presigned PUT/GET to DigitalOcean Spaces, private bucket, RBAC-gated downloads, FINTRAC-aware retention. Inert until `SPACES_*` set. Model `platform_application_documents` (migration 042).
- E-sign via SignNow (`services/esign/signnow_adapter.py` + OAuth) ‚Äî send-for-signature, status poll, HMAC webhook. `agreement_status` state machine.

**Notifications (email/SMS/in-app):**
- Templates: 10 email (`app/templates/emails/`: application approved/rejected/under_review, adverse_action_notice, documents_required, monthly_statement, payment_due_reminder, payment_overdue, payment_reminder, urgent_action_required) + 5 SMS.
- Dispatch: mock vs real by `USE_REAL_NOTIFICATIONS`. Email via SendGrid (default) or Resend. SMS via Twilio.

- **Adverse-action notice** (`adverse_action.py`) ‚Äî Canada-specific decline notice (PIPEDA + provincial): principal reasons, CRA disclosure, non-discrimination notice.

**Reporting/analytics:** `/admin/analytics/*` (portfolio, vintage, originations, CEI + 7 report types); clinic dashboards; Prometheus `/metrics`.

## 6. Audit, Maker-Checker, RBAC, Compliance

- **Event-sourced audit trail:** `platform_events` (append-only; actor, JSONB payload, correlation_id). Backs audit log, dunning idempotency, maker-checker, notification dedup, webhook replay checks.
- **Maker-checker:** (1) charge_off / disburse require a *different* admin to approve (unique terminal via migration 039); (2) vendor profile change requests (migrations 036/037).
- **RBAC:** UserRole = admin / staff / patient / vendor; Role‚ÄìPermission‚ÄìRolePermission model; `require_roles` / `require_permission` dependencies; RLS auth middleware sets DB row-level-security context. Clinic surface scoped by vendor_id; applicant by patient JWT.
- **Consent management:** `platform_consents` ‚Äî 9 purposes; WORM (immutable text + version, IP/UA captured, revocable).
- **PII/security:** SIN only as encrypted Fernet token (+ `sin_last3`); vendor log redaction; secrets encrypted at rest; rate limiting; CSRF; security headers; docs disabled in prod.
- **Turnkey migration path:** `PlatformLoan.source='turnkey_migration'` + `legacy_account_number` (idempotent re-import, migration 035) ‚Äî legacy loans importable without a PaySpyre application.
